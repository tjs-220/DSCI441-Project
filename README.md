This project evaluates the trade‑offs between Support Vector Machines (SVMs) and Random Forests (RFs) for classification tasks on tabular data, with the goal of providing practical, statistically grounded guidance for model selection. The analysis focuses on how these two classical machine learning algorithms differ in accuracy, robustness to noise, sensitivity to class imbalance, scalability, training time, and dependence on preprocessing. To support this comparison, the project implements a fully reproducible pipeline that loads and cleans the data, encodes categorical variables, scales numerical features, and performs a stratified train/test split. Both SVM and Random Forest models are trained using scikit‑learn pipelines with integrated preprocessing, and each model undergoes hyperparameter tuning via GridSearchCV. Model performance is evaluated using accuracy, classification reports, and confusion matrices, with visualizations generated through seaborn and matplotlib.

The dataset used in this project is the UCI Adult Income (Census) Dataset, retrieved programmatically using the ucimlrepo Python package. This dataset contains demographic and employment‑related attributes used to predict whether an individual earns more than $50K per year. Its combination of numerical and categorical features, along with inherent class imbalance, makes it a strong benchmark for evaluating classical machine learning models and preprocessing workflows.

To run the project, you will need Python along with several required packages, including numpy, pandas, scikit-learn, matplotlib, seaborn, and ucimlrepo. These can be installed individually or collectively using the requirements.txt file. Once the dependencies are installed, you can execute the project by running either train_svm.py or train_rf.py. Each script automatically loads the dataset, builds the preprocessing pipeline, splits the data, performs hyperparameter tuning, trains the model, and evaluates it on the test set. Running the scripts will print evaluation metrics to the console and display a confusion matrix plot for the selected model. The preprocessing logic is contained in preprocessing.py, while evaluation utilities are defined in evaluation.py, making the project modular and easy to extend.
