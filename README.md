Evaluating the Trade‑offs Between SVM and Random Forests for Tabular Data

This project investigates the comparative strengths and limitations of Support Vector Machines (SVMs) and Random Forests (RFs) when applied to tabular data, focusing on how these two widely used classical machine learning algorithms differ in accuracy, scalability, training time, memory usage, robustness to noise, and sensitivity to feature transformations. The goal is to provide practical guidance for choosing between the two methods in real‑world machine learning scenarios where deep learning is unnecessary or impractical. To accomplish this, the project implements a reproducible experimental pipeline that trains and evaluates both models on the UCI Adult Income dataset, examining how each algorithm responds to changes in dataset size, dimensionality, and class imbalance.

The methodology includes a full preprocessing workflow—cleaning the data, encoding categorical variables, normalizing numerical features when appropriate, and creating stratified train/test splits—followed by baseline model training, hyperparameter tuning, and systematic evaluation using metrics such as accuracy, precision, recall, F1‑score, and confusion matrices. Additional experiments assess robustness by introducing controlled noise, altering class distributions, and testing the impact of feature scaling, feature selection, and engineered transformations. The study also measures computational efficiency by tracking training time, inference time, and memory usage as dataset size increases. Results are visualized through learning curves, ROC curves, feature importance plots, and other diagnostic charts, culminating in an analysis that clarifies when and why SVMs or Random Forests may be preferable for a given tabular learning task.

The dataset used in this project is the UCI Adult Income (Census) Dataset, available at https://archive.ics.uci.edu/dataset/2/adult, which contains demographic and employment‑related attributes used to predict whether an individual earns more than $50K per year. This dataset’s mix of categorical and numerical features makes it well‑suited for evaluating classical machine learning models and the preprocessing steps they require.

References for this work include Dua and Graff’s UCI Machine Learning Repository documentation for the Adult dataset; Breiman’s foundational 2001 paper on Random Forests; Cortes and Vapnik’s 1995 work introducing Support Vector Machines; and Pedregosa et al.’s 2011 paper describing the scikit‑learn library used to implement the models and experimental pipeline.
